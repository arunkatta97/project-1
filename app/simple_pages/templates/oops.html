Ooject Oriented Design

The technique of planning a system of interacting objects for the goal of addressing a software problem is known as object-oriented design. It's one method of software development.

An object is a collection of enclosed data and processes that comprise a single entity. The 'object interface' specifies how to interact with the object. The interaction of these items describes an object-oriented application. Object-oriented design is the process of creating objects and their interactions in order to solve a problem that has been recognized and documented through object-oriented analysis.

Encapsulation

One of the core principles in object-oriented programming is encapsulation (OOP). It refers to the concept of encapsulating data and the functions that operate with it in a single unit, such as a Java class. This idea is frequently used to hide an object's internal state representation from the outside world.

Abstraction

Abstraction is the process of describing crucial and unique traits without incorporating any background information or explanation. Data abstraction makes database design easier.

Polymorphism

Data polymorphism refers to the ability of data to be processed in multiple ways. It allows you to complete the same activity in a variety of ways. It includes of method overloading and overriding, which entails writing a method once and executing several jobs with the same name.

Inheritance

Inheritance refers to a class's ability to take on the capabilities or properties of another class, referred to as the parent class. We inherit properties from other classes when we write a class. As a result, when we build a class, we don't have to write all of the properties and functions again because they can be inherited from another class. Inheritance allows the user to reuse code and reduce redundancy whenever possible.
