SOLID

SOLID is a mnemonic acronym representing five design principles in software engineering that aim to make software designs more clear, adaptable, and maintainable. The principles are a subset of several that American software engineers support.

The SOLID ideas are:

  • "There should never be more than one reason for a class to change," says the single-responsibility concept. To put it another way, each class should only have one task.
  • "Software entities should be open for extension but closed for modification," says the openâ€“closed principle.
  • "Functions that employ pointers or references to base classes must be able to use objects of derived classes without knowing it," according to the Liskov substitution principle.
  • "Many client-specific interfaces are preferable than one general-purpose interface," says the interface segregation concept.
  • "Rely on abstractions, not concretions," says the dependency inversion principle.
